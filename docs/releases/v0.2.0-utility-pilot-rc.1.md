## Included Read-Only Flow

- Manual wallet-address input.
- Ethereum address validation.
- Read-only Sepolia chain ID check through allowlisted RPC endpoints.
- Read-only `balanceOf(wallet)` call against the canonical TIDE token.
- Read-only `decimals()` call for balance formatting.
- `Promise.allSettled` fallback across RPC endpoints.
- Live, Cached, Stale, Wrong Chain, and Unavailable data states.
- Optional off-chain message proof text for future manual review design.

No wallet connection is required for browsing. The flow does not request approvals, transfers, fees, private keys, seed phrases, emails, guest data, or booking confirmation.
