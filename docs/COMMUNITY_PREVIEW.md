# TikiDeco Community Preview

Status: public Sepolia prototype preview.

This preview is designed to collect feedback on public documentation, verification flows, site clarity, and conceptual utility scenarios. It is not a token sale, product launch, mainnet launch, hospitality launch, or fundraising campaign.

## Preview Boundary

TikiDeco / TIDE is currently:

- an open-source Ethereum Sepolia prototype;
- a fixed-supply testnet token with verified Sepolia source code;
- a read-only public site and documentation set;
- a Safe-controlled canonical V1 Sepolia deployment;
- a V2 candidate codebase for future review.

TikiDeco / TIDE is not currently:

- deployed on mainnet;
- offered for sale;
- assigned any stated monetary value;
- independently audited;
- connected to active hotel benefits;
- connected to equity, debt, revenue, property, or return rights.

## What Feedback Is Useful

Useful feedback should focus on whether a public viewer can understand and verify the project without needing private context.

High-value feedback areas:

- Is the website clear within the first 10 seconds?
- Are the no-sale, no-value, no-mainnet, and no-independent-audit boundaries visible?
- Can a non-technical reader understand what Sepolia means?
- Can a technical reader verify the token, vault, Safe, reports, and manifest?
- Are the utility scenarios clearly conceptual or planned, not active benefits?
- Are any words confusing, too promotional, or legally risky?
- Is the current English copy clear enough to serve as the reviewed source for any future translation?
- Are the docs easy to navigate from GitHub and the public site?

## What Feedback Is Out Of Scope

The preview is not a request for:

- token purchase inquiries;
- price or listing discussion;
- mainnet deployment pressure;
- promises of hospitality benefits;
- partnership announcements;
- private-key, wallet-signing, or transaction requests;
- legal conclusions from non-counsel reviewers.

## Response Standard

Maintainers should answer preview feedback with:

- a direct acknowledgement;
- a link to the relevant source of truth;
- a clear distinction between current, planned, and conceptual;
- no sale, price, listing, or return language;
- an issue or PR link when a change is needed.
