# First Future Benefit Specification

Status: proposed and inactive. This specification does not create availability, acceptance, booking, or a real-world entitlement.

## Early RSVP Eligibility For A Community Preview

| Field | Current decision |
| --- | --- |
| Operator | Not established. A named operator with authority and documented responsibilities is required before activation. |
| Inventory | `0` available. A later approved campaign must publish a small fixed capacity. |
| Qualification rule | Proposed read-only Sepolia balance threshold plus published snapshot or approved live-check window and manual review. Meeting the threshold does not confirm access. |
| Availability | Not available. The campaign remains `draft-not-live`. |
| Blackout conditions | All dates remain unavailable until venue, staffing, safety, legal, privacy, and operations reviews are complete. Later rules may add weather, capacity, venue, or local-law blackout conditions. |
| Support owner | Unassigned. A named support owner and response window are required before activation. |
| Dispute process | Use the documented manual review and dispute flow only after an approved request window is published. Aggregate outcomes only. |
| Cancellation | The operator may cancel for legal, safety, staffing, venue, weather, capacity, or operational reasons under published rules. No cash redemption or substitute is promised. |
| Legal review status | `not-approved`; counsel review required. |

The example is non-financial, limited, conditional, subject to availability, manually reviewed, non-transferable as a guest right, and not redeemable for cash.
