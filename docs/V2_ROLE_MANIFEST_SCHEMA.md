# TikiDeco V2 Role Manifest Schema

Status: schema for non-canonical V2 candidate deployments.

The V2 deployment script writes `deployments/<network>-v2-roles.json` after deployment and on-chain role assertions. The manifest is not a canonical deployment record and must not replace `deployments/canonical.json`.

## Schema Identifier

`tikideco.v2.role-manifest/1`

## Required Fields

| Field | Type | Description |
| --- | --- | --- |
| `schema` | string | Must equal `tikideco.v2.role-manifest/1`. |
| `version` | string | V2 candidate version label. |
| `network` | string | Hardhat network name. |
| `chainId` | number | Chain ID returned by the provider. |
| `deployedAt` | string | ISO timestamp generated by the script. |
| `deployer` | address | Address that submitted the deployment transactions. |
| `defaultAdminDelaySeconds` | number | Delay used by `AccessControlDefaultAdminRules`. |
| `contracts.token` | address | V2 candidate token address. |
| `contracts.vestingVault` | address | V2 candidate vesting vault address. |
| `roles.token.DEFAULT_ADMIN_ROLE` | address[] | Expected token default admin holder list. |
| `roles.token.PAUSER_ROLE` | address[] | Expected token pauser holder list. |
| `roles.token.REPORTER_ROLE` | address[] | Expected token reporter holder list. |
| `roles.vestingVault.DEFAULT_ADMIN_ROLE` | address[] | Expected vault default admin holder list. |
| `roles.vestingVault.VESTING_ADMIN_ROLE` | address[] | Expected vault vesting admin holder list. |
| `treasury` | address | V2 treasury address. |
| `metadata.projectName` | string | Neutral bounded project name. |
| `metadata.businessEntity` | string | Bounded business entity string. |
| `metadata.jurisdiction` | string | Bounded jurisdiction string. |
| `metadata.projectURI` | string | Bounded project URI string. |

## Post-Deployment Assertions

The deployment script must fail if:

- any expected role holder does not hold the role on-chain;
- the deployer holds a privileged role that is not expected by the role configuration;
- public-network deployment lacks `CONFIRM_NON_CANONICAL_V2_DEPLOY=true`;
- public-network deployment omits any V2 privileged address.
