# Hospitality Operations Gate

Status: `not approved`

This document is an operations-readiness control. It is not legal advice, not legal approval, not a booking system, not a value statement, and not an offer.

No active guest utility should be announced or operated until every item below is defined, reviewed, and approved.

## Required Before Active Guest Utility

| Area | Required definition | Current status |
| --- | --- | --- |
| Property/operator | Identify the property, venue, operator, responsible entity, and authority to offer the pilot | not approved |
| Available inventory | Define exact inventory, capacity, dates, blackout dates, and allocation method | not approved |
| Perk policy | Define non-cash perk scope, limits, restrictions, cancellation, and no-resale rules | requires review |
| Cancellation policy | Define cancellation reasons, participant notice, rescheduling, and reporting process | requires review |
| Staff process | Define staff roles, manual review, override authority, escalation, and logging | requires review |
| Support process | Define support channel, response time, dispute intake, and correction workflow | requires review |
| Dispute process | Define dispute deadline, evidence, reviewer, decision authority, and finality | requires review |
| User terms | Publish user-facing pilot terms before collecting data or accepting requests | not approved |
| Privacy process | Define data minimization, consent, retention, access control, deletion, and incident handling | requires review |
| Local legal constraints | Review local hospitality, consumer, privacy, tax, sanctions, and venue constraints | requires review |

## No-Go Boundaries

- No active hotel benefit is live.
- No guest booking or reservation is confirmed through TIDE.
- No cash redemption is available.
- No resale value is represented.
- No transfer of guest rights is supported.
- No guaranteed benefit is promised.

## Required Command

```bash
npm run mainnet:check
```

This command must remain blocked until hospitality operations are approved alongside legal, security, treasury, privacy, tax, and communications gates.
