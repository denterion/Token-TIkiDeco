# Operator Interview Guide

Use these questions to test whether the sandbox reflects an understandable hospitality workflow. Do not request guest records, booking exports, internal credentials, or sensitive commercial data.

1. Which campaign decision is hardest for staff to explain consistently today?
2. Who is allowed to define rules, inventory, exceptions, and stop conditions?
3. What minimum evidence should staff see before approving a request?
4. How should duplicate or disputed requests be handled?
5. Which inventory changes require a second reviewer?
6. What conditions should pause or close a campaign immediately?
7. Which aggregate outcomes would be useful in a transparency report?
8. Which data must never appear in public reports or on-chain records?
9. What support owner and escalation path would be required for a limited pilot?
10. What legal, privacy, security, and operations evidence would your organization require before testing with real users?

The interview is discovery only. It does not create a partnership, commitment, active benefit, or production approval.
