# X Community Preview Posts

Status: draft public copy. These posts are for a Sepolia prototype community preview. They do not announce an offering, mainnet deployment, active guest benefits, or independent audit completion.

## 1. Pinned Post

TikiDeco / TIDE is an open-source Ethereum Sepolia prototype exploring transparent hospitality-linked token infrastructure.

Current scope: read-only website, public docs, verified testnet contracts, Safe governance notes, and cautious utility-pilot planning.

Feedback: https://github.com/denterion/Token-TIkiDeco/issues

## 2. Release Candidate Post

v0.2 utility-pilot RC work is focused on read-only verification:

- manual wallet address input
- Sepolia `balanceOf` read
- cached/stale/unavailable RPC states
- draft-not-live campaign gate
- public blocker register

No wallet transaction flow is included.

## 4. Transparency Post

TikiDeco keeps project claims tied to public evidence:

- canonical deployment manifest
- verified Sepolia source links
- project facts
- claims matrix
- report hashes
- release gates

If a statement is not verifiable, it should not be public copy.

## 5. Utility Pilot Boundary Post

The utility-pilot work is still draft-not-live.

The current flow is read-only and testnet-only. It can check a Sepolia TIDE balance for review design, but it does not create booking confirmation, payment flow, or active hospitality access.

## 6. Safe Governance Post

TikiDeco documents privileged Sepolia V1 control through a Safe and public manifests.

The goal is simple: public readers should be able to check who controls sensitive actions, where treasury is listed, and which contracts are canonical.

## 7. Report Hash Post

Public reports are more useful when readers can verify them.

TikiDeco tracks report files, SHA-256 hashes, manifest references, and Sepolia transaction links where available.

That makes corrections and status changes easier to review.
