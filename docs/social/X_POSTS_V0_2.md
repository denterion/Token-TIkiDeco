# X Posts For v0.2 Utility Pilot

Status: draft social copy. This is not legal advice, not a sale, not a value statement, not a mainnet announcement, and not an independent audit claim.

## 1. v0.2 Utility Pilot

TikiDeco v0.2.0-utility-pilot is a Sepolia-only pre-release for a read-only eligibility flow.

TIDE is not offered for sale, has no stated monetary value, is not deployed on mainnet, and has not completed an independent audit.

## 2. No-Sale Boundary

The TIDE pilot flow is read-only: no wallet connection required for browsing, no transaction signing, no approvals, no transfers, no fees, and no purchase flow.

It is a Sepolia prototype, not a token sale.

## 3. Read-Only Verification

v0.2 adds a manual wallet-address checker that reads Sepolia `balanceOf(wallet)` from the canonical TIDE token through allowlisted RPC endpoints.

If RPC data is unavailable, the UI shows that state instead of inventing a balance.

## 4. Report Hashes

TikiDeco uses public docs, canonical manifests, and report hashes to keep project status reviewable.

The goal is transparent records, not price, sale, listing, or mainnet claims.

## 5. Safe Governance

The canonical Sepolia V1 owner is documented as a Safe. That does not make the project production-ready, but it improves operational discipline for the prototype.

V2 remains candidate code only.

## 6. Why Mainnet Is Blocked

Mainnet is not approved.

The gate stays blocked until legal, audit, treasury, privacy, operations, guest terms, public communications, and token distribution reviews are complete.

## 7. Why Independent Audit Is Not Claimed

Internal review, tests, Slither, Foundry, and CI are useful preparation. They are not an independent audit.

TikiDeco will not describe V2 as independently audited unless a real external audit exists and is published with scope and findings.

## 8. Community Feedback

Community preview feedback is welcome on clarity, docs, translations, accessibility, utility boundaries, and verification.

Please do not frame TIDE as a sale, investment, price, listing, mainnet, or active hospitality benefit.

## 9. Allocation Policy

The testnet allocation policy is Sepolia-only and draft-gated.

Any future campaign needs rules, caps, manual review, privacy-safe reporting, and approval gates before it can be treated as published.
