# Reviewer Selection Checklist

Use before sharing the immutable V2 review package.

- [ ] Solidity review history is verifiable.
- [ ] Comparable past work is public or referenceable.
- [ ] Reviewer is independent from TikiDeco token allocation, treasury and deployment decisions.
- [ ] Compensation is not tied to listing, market activity, token distribution or a favorable conclusion.
- [ ] Reviewer will not request private keys, seed phrases or signer recovery material.
- [ ] Methodology includes manual review, tests, static-analysis triage and deployment-guard review.
- [ ] Written findings include severity, file/line, scenario, impact, likelihood and remediation.
- [ ] Reviewer is available for a scoped retest.
- [ ] Disclosure and publication policy is documented.
- [ ] Conflicts of interest are disclosed in writing.
- [ ] Confidentiality boundaries and incident contact are agreed.
- [ ] Exact candidate commit and package checksum are acknowledged before work starts.

Selection does not change V2's non-canonical, non-deployed status.
