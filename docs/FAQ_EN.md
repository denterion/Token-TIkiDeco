# TikiDeco FAQ

## What is TikiDeco / TIDE?

TikiDeco / TIDE is an open-source Ethereum Sepolia prototype exploring transparent loyalty and access infrastructure for a future hospitality concept.

## Is TIDE deployed on mainnet?

No. The current canonical deployment is on Ethereum Sepolia, which is a test network.

## Is TIDE offered for sale?

No. TIDE is not offered for sale and has no stated monetary value.

## What currently works?

The current Sepolia deployment includes a fixed-supply ERC-20-style token, verified source code, owner-controlled pause logic, report hash publishing, a vesting vault, Safe-controlled privileged operations, and a public read-only dashboard.

## What does not exist yet?

There is no mainnet deployment, token sale, exchange listing claim, independent audit, active hotel benefit, confirmed hotel partnership, or completed property claim.

## Who controls privileged operations?

The canonical Sepolia V1 token and vault are controlled by the owner Safe listed in the deployment manifest. The current Safe threshold is documented as `3-of-3`.

## Has the project been audited?

No. The repository includes tests and internal review materials, but an independent smart-contract audit has not started.

## What can people verify?

People can verify the canonical deployment manifest, Sepolia Etherscan source-code pages, repository docs, test suite, public reports, and read-only dashboard.

## Are hospitality perks active today?

No. Loyalty, access, and hospitality-related use cases are research directions. They are not active guest benefits or commitments.

## Does TIDE represent ownership in a hotel?

No. The current prototype does not provide hotel ownership, equity, debt, revenue rights, or other financial rights.
