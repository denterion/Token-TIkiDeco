Conceptual hospitality layer

A Miami Beach idea with disciplined public infrastructure.

TikiDeco combines art deco geometry, coastal leisure, and operational transparency. The current work is not a completed hotel product; it is a public testnet prototype for researching how a hospitality brand concept could document rules, allocations, reports, and access programs on-chain.

The visual language is hospitality-first: calm surfaces, clear records, and a restrained neon signal for testnet status and verification.

Security and governance

Controls before promotion.

Safe ownership

Admin control for the Sepolia legacy deployment sits behind a 3-of-3 Safe. This is testnet governance discipline, not a mainnet approval.

Hardhat 3 tooling

The project uses Hardhat 3, built-in coverage and gas statistics, npm audit checks, Slither triage, and dependency monitoring.

V2 candidate boundary

OpenZeppelin V2 remains review code unless a later canonical manifest explicitly promotes it. It is not described as audited.

Risks and disclaimers

Plain boundaries for a public prototype.

TIDE is a Sepolia testnet prototype. Sepolia tokens are testing artifacts and have no stated monetary value. The project is not offering token sales, hotel ownership, revenue rights, or completed hospitality services.

Possible loyalty, access, and reporting workflows are research directions. They should not be read as active benefits or commitments.
